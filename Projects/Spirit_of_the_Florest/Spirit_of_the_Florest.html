Project Details
Project Type Uni/Grouped work
Project Role(s) Lead Programer
Duriation 6 Mouths

Project Crawler is a 3rd person rogue-like dungeon crawler game, where your health is also your currency and ability resource. creating a risk vs reward gameplay. Equip yourself with new abilities and upgrade them at the cost of your health. Project Crawler was created for the university of Portsmouth game jam with a team of 6 members, with the game’s topic being ‘a dungeon crawler with a single resource’.

Key Contributions
Character Controller


I developed the character controller system to allow for freedom of movement and give as much control to the player allowing for quick and precise movement to avoid incoming attacks

Combat System

I developed the player to have two states for both melee (or walking) and ranged, as each need their own player rotation system and animation controller. I created these systems to be fast and fluid to avoid interrupting the flow of the game.


Ability System


I create a large set of systems for unlocking, equipping, using abilities and handling upgrades. All these systems work together to make the flow of acquiring and using abilities as streamlined as possible, to not slow down the pace of gameplay.

modual


I designed a modular ability system that contains the majority of an ability’s logic (Cooldowns, Cost, Casting, Charge handler, Checks, ect) and can be inherited by an ability script, allowing for simple implementation of abilities and project wide alterations to the logic system.

Project Devlog

Introduction

For the beginning of the new project, we were tasked with crating a game from a of list options:

• Starship bridge simulator
• University tycoon
• Dungeon crawler with one resource
• A game suitable for someone visually impaired
• 2d platformer within a 3d world
• Lockdown – the game
• A game using an unusual control device

Once the team had formed, we researched the options and discussed potential gameplay loops, ideas and mechanicals for each option. This discueion narrowed down list and the team seemed to be mainly interested in the dungeon crawler idea, including myself. This discussed formed the main gameplay loop of having the player with non-regenerating health and no health cap, but by killing enemies they would drop health that can be picked up. The health could be spent on potential abilities, items and upgrades as each level would increase the difficulty as well as adding a sense of risk and reward to the players choices. There were still some key decisions that the team had not decided on yet such as the player perspective, either being top down, isometrics or third person. The team easily decided to use unity for this project as we used it for the previous project and the team was already famailiter using the software. I was assigned as a lead programmer for the project with 1 other programmer. I was eager to start on the project but with designs and ideas still undecided, I begin to envision general mechanicals and develop a very primitive version of them.

1st Feb 2021 - 1st Feb 2021
basic systems and proof of concepts

Primitive Skill Tree

I began developing a skill tree system using unity’s UI system, this system would be the most likely way of obtaining new abilities in the game. I constructed a skill tree node script so that in order to unlock a node, there will need to be enough connected parent nodes unlocked, later development on the system will requiring the player to have enough health to unlock them, once the health system is setup. I also made the UI element go red, yellow or green depending on their status as well as adding a UI text element displaying how many parent nodes are required to be unlocked.
